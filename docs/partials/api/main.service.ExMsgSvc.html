ExMsgSvc
service in module main

Description

Handles maintaining data for the message related views.

Dependencies

  • $scope
  • Data

Events

  • countCleared

    This message is fired when we call the clearMsgArray function. Let's listeners know that the property numType has changed.

    Type:

    broadcast

  • msgAvailable

    Event to let observers know that message data is available. Fires only if there are messages in the stream.

    Type:

    broadcast
